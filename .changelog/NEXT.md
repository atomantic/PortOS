# Unreleased Changes

## Added

- **Writers Room — tabbed sidebar (Characters | World | Scenes | Boards | Config), top progress strip, and "Stop all" cancel for in-flight scene renders.** The right column was a single-purpose Storyboard pane with the bibles hidden behind drawer overlays from the work overflow menu — clicking "Refresh Characters" closed the menu and hid the only spinner, so users couldn't tell whether anything was happening. Replaced with a 5-tab sub-nav: **Characters** and **World** render their bibles inline with a top "Refresh from prose" button (with running spinner per-button); **Scenes** is the prose-derived outline (clickable rows jump to that segment in the textarea); **Boards** keeps the existing storyboard scene cards; **Config** consolidates the gear-icon's Adapt LLM picker, World style, and image-gen settings. A status strip ("Refreshing characters…") sits below the tab nav whenever any analysis runs — visible from any tab even after the overflow menu closes. Each tab's icon shows a spinner when its kind is the running one. Adapt auto-jumps to the Boards tab when it starts. Long queues of auto-generated scene images can now be aborted: when one or more scenes are rendering, a banner appears above the cards ("Rendering N scenes…") with a red **Stop all** button — clicking it POSTs `/api/image-gen/cancel { all: true }` (queued cancel before running so the slot doesn't refill mid-loop), then iterates each running scene's imperative ref and calls `cancel()` to roll local state back to idle (idempotent — handles the race where the cancellation `image-gen:failed` socket event arrives first and flips the card to `error`). Per-card unmount cleanup in `onRunningChange` prevents stale entries inflating the count if a scene disappears mid-render. Touched: `client/src/components/writers-room/{StoryboardPanel,WorkEditor,SceneCard,CharactersBible,SettingsBible}.jsx`, `client/src/services/apiImageVideo.js`, `server/routes/imageGen.{js,test.js}`.

- **Writers Room — 3-step storyboard setup (Characters → Settings → Adapt) replaces the lone "Run Adapt" CTA.** A fresh story used to greet the user with a single "Run Adapt" button, but Adapt's quality depends on the character + setting bibles being populated first (otherwise the LLM re-improvises descriptions every scene, breaking visual continuity). The empty-state CTA now shows a guided 3-step panel: each step has a status pill (numbered → spinner → green check), its own `Run characters` / `Run settings` / `Run Adapt` button, and a prominent `Run all in order →` master action that fires the sequential pipeline (`runFullPipeline`) so the user can one-click the whole flow. Once Adapt has run, a separate `BiblesMissingNotice` banner shows above the scenes if either bible is empty, with inline "Extract characters" / "Extract settings" buttons — so users who ran Adapt before populating bibles get a clear path to fix the visual drift without leaving the panel. `runAnalysis` now returns success/failure so the pipeline can bail on first error rather than silently chaining through a broken state. Touched: `client/src/components/writers-room/{StoryboardPanel,WorkEditor}.jsx`.

CURRY OCITY — NIGHT`) with `description`, `palette`, `era`, `weather`, `recurringDetails`, plus prose evidence + missing-fields hints. New stage `writers-room-settings` (md prompt + stage-config seed); new service `server/services/writersRoom/settings.js`; new routes `GET/POST /api/writers-room/works/:id/settings`, `PATCH/DELETE /api/writers-room/works/:id/settings/:settingId`; new `SettingsBible.jsx` drawer in WorkEditor with create/edit/delete; new `Refresh settings` menu item. Sluglines normalize identically on client + server (uppercase, em-dash → space, punctuation stripped) so `INT. KITCHEN — NIGHT` and `INT KITCHEN - NIGHT` resolve to the same entry. The script (Adapt) prompt now also receives both bibles as variables (`existingCharactersJson`, `existingSettingsJson`) and is instructed to (a) cite canonical character `physicalDescription` rather than re-improvising and (b) reuse the bible's slugline string verbatim so the storyboard pipeline auto-matches. SceneCard's image-prompt composition now layers the matched setting between world-style and characters: `{worldStyle} . {workTitle} . {setting.description} {setting.palette} {setting.recurring} . Featuring — {chars} . {scene.visualPrompt}` (priority order: visualPrompt always survives, then setting baseline, then characters drop one-by-one to fit the 1900-char budget). End-to-end verified — gemini-2.5-pro extracted 5 locations from a 4,800-word draft in 66s, merged with the user-edited entry, and now feeds image gen for visual continuity across scenes set in the same place. New files: `server/services/writersRoom/settings.js`, `client/src/components/writers-room/SettingsBible.jsx`, `data{,sample}/prompts/stages/writers-room-settings.md`. - **Writers Room — Codex / Claude Code CLI support for AI passes, and tail-truncated CLI errors so failures are actually readable.** `callCliProvider` previously branched on `gemini-cli` and treated everything else as "spawn the binary, pipe prompt to stdin" — which Codex and Claude Code reject because they need a subcommand to run non-interactively. Picking Codex as the Adapt LLM exited code 1 with `Error: stdin is not a terminal` because `codex` defaulted to its REPL; Claude Code would have the same gap. Replaced the binary `isGeminiCli` switch with a `buildCliInvocation(provider, model)` dispatcher mirroring `runner.js` / `agentCliSpawning.js`: `codex` → `exec [--model <m>] -`, `claude-code` / `claude-code-bedrock` → `-p - [--model <m>]`, `gemini-cli` → unchanged (`--prompt` arg + `stdio[0] = 'ignore'`), default → spawn binary and pipe prompt to stdin. Also flipped CLI error truncation from `slice(0, 500)` → `slice(-2000)` since the actual failure (e.g. `"The 'gpt-5' model is not supported when using Codex with a ChatGPT account"`) lives at the end of stderr — the head-truncation was clipping the banner and hiding the real cause. Touched: `server/services/writersRoom/evaluator.js`.

- **Writers Room — per-stage provider/model picker in Storyboard settings, and surfaced Adapt failures.** The storyboard's gear-icon settings drawer now exposes an **Adapt LLM** picker with the same Tier-vs-Specific toggle the global Prompt Manager uses (`Tier: default/quick/coding/heavy` or `Specific: <provider> + <model>`). Picking a value PUTs `/api/prompts/writers-room-script` so the same selector works from `/prompts` or any panel that runs the stage. The evaluator's `callAI` now reads `stage.provider` + `stage.model` first (falling back to active provider only when no override is set) and resolves tier names through the provider's `defaultModel`/`lightModel`/`mediumModel`/`heavyModel`, fixing a case where every Writers Room AI pass silently routed through `getActiveProvider()` regardless of stage config. The Storyboard panel also now surfaces failed runs: instead of silently falling back to "No storyboard yet" when the latest Adapt snapshot is `failed`, it shows an error banner with the actual error message, an `Adjust LLM` shortcut into the settings drawer, and a `Re-run Adapt` button. (The 5-minute Gemini-CLI timeout that prompted this — `CLI AI call timed out after 300000ms` on a 4,800-word draft — was the original symptom: the call had been failing on disk for hours but the UI just looked empty.) New file: `client/src/components/writers-room/StagePromptModelPicker.jsx`. Touched: `server/services/writersRoom/evaluator.js`, `client/src/components/writers-room/StoryboardPanel.jsx`.

## Changed

- **Gemini-CLI provider model list updated to current Google releases.** Replaced the deprecated `gemini-2.5-flash-lite` with the current set: `gemini-3.1-pro` (heavy), `gemini-3-flash` (medium), `gemini-2.5-pro`, `gemini-2.5-flash` (light + default). Tier mappings remap: `defaultModel`/`lightModel` → `gemini-2.5-flash`, `mediumModel` → `gemini-3-flash`, `heavyModel` → `gemini-3.1-pro`. The hardcoded gemini-cli fallback in `brain.js` and `writersRoom/evaluator.js` (used when both `defaultModel` and `lightModel` are unset on the provider) now points to `gemini-2.5-flash` instead of `gemini-2.5-flash-lite`. Mirror change applied to the upstream `portos-ai-toolkit` `providers.sample.json` so fresh installs pick up the new defaults. Touched: `data{,sample}/providers.json`, `server/services/{brain.js,brain.test.js,writersRoom/evaluator.js}`, `../portos-ai-toolkit/src/defaults/providers.sample.json`.

- **Writers Room — Adapt prompt rewritten for paragraph-grain output.** The Adapt stage was producing 8 scenes for a 4,800-word piece because the prompt told the LLM "A scene = one continuous unit of action in one location" + "Don't pad. If the natural arc is 3 scenes, return 3." That's a screenplay rule; wrong for a storyboard companion where dense beat-level cards are the whole point. Reframed "scene" as "one storyboard panel," set an explicit count anchor ("a 4,000-word piece with ~80 paragraphs should yield ~50–80 scenes, not 8"), listed when to SPLIT (default — every substantive paragraph; new visual framing; back-and-forth dialogue) vs MERGE (rare — one-sentence paragraphs depicting the same instant; pure dialogue tags), removed the "don't pad" anti-pattern, and tightened action/visualPrompt budgets since each work now has many more cards. Sluglines may repeat across consecutive same-location scenes. Touched: `data{,sample}/prompts/stages/writers-room-script.md`.

- **Writers Room — analysis storage collapsed from history-of-runs to latest-per-kind.** The Adapt / Evaluate / Format / Characters analyses were each accumulating a fresh `wr-analysis-<uuid>.json` file per run, growing forever. The user only ever cares about the latest of each kind. Analysis id is now just the kind (`script`, `format`, `evaluate`, `characters`); files are `<kind>.json`; re-running a kind atomicWrite-overwrites its previous snapshot. Boot-time `recoverStuckAnalyses` gained a one-time migration that scans for legacy UUID-named files, groups by kind, keeps the latest per kind (by `completedAt|createdAt`), rewrites as `<kind>.json`, and deletes the legacy files. Idempotent. Touched: `server/services/writersRoom/evaluator.js`.

- **Writers Room — graphic novel + comic book presets suppress text/word-balloons.** Fast image models (FLUX.2 Klein, Schnell) render text as garbled gibberish. The `graphic-novel` and `comic-book` presets were inviting it via "sequential art panels" / generic comic-cover language, producing scenes with baked-in title-card text and word balloons. Both presets now drop the multi-panel language, add `single full-bleed image, no text` to the prompt, and aggressively negative-prompt `text, words, letters, lettering, typography, title, logo, word balloon, speech bubble, caption box, sound effect`, plus `onomatopoeia + comic book cover` for comic book. Existing works that already saved an old preset prompt need to re-pick the preset from the dropdown to refresh their stored text; custom-edited prompts are left alone. Touched: `server/lib/writersRoomStylePresets.js`.

## Fixed

- **Writers Room mobile editor squashed to ~150px (regression).** `WritersRoom`'s `<main>` element had `min-h-0 flex flex-col` with no `flex-1`, so on mobile (below `md:`, where the parent flips to `display: grid`) the main area sized to its natural content height — collapsing the textarea to whatever space was left after the library, header, and tab strip. Added `flex-1` so main grows to fill the remaining viewport on mobile; desktop is unaffected because `md:grid` takes over and the `flex-1` is dormant. Touched: `client/src/pages/WritersRoom.jsx`.

- **Collapsed sidebar footer overflow — theme toggle + bell wouldn't fit in the 64px rail.** The footer's two visible icons (ThemeModeToggle ~30px and NotificationDropdown bell ~36px plus a 2px focus-ring offset) summed to ~70–80px side-by-side, blowing out a `lg:w-16` collapsed sidebar that has only 32px of content width after `p-4`. Reduced collapsed-state footer padding to `p-2` (lg only) and stacked the buttons vertically (`lg:flex-col`) so each icon centers in the rail. Expanded layout is unchanged. Touched: `client/src/components/Layout.jsx`.

- **Writers Room mobile prose area squeezed by AI panel.** On mobile/tablet (below `lg`), the editor and AI sidebar stack vertically; the AI sidebar `<aside>` had no flex-grow, so it expanded to its (long) content height — leaving the prose textarea with whatever was left, often only ~150px. Gave the textarea wrapper `flex-[2]` and the AI aside `flex-1` on mobile (both reset at `lg:` where the desktop fixed-width split takes over) so the prose now gets ~2/3 of the available editor height and the AI panel scrolls internally within its 1/3. Touched: `client/src/components/writers-room/WorkEditor.jsx`.

- **Writers Room loaded empty in dev (Vite on `:5554`) while production worked.** All five `mountedRef` guards in the Writers Room and Ask pages used `useRef(true)` plus a cleanup-only `useEffect(() => () => { mountedRef.current = false; }, [])` to skip post-`await` `setState` after unmount. In React 18 `<StrictMode>` (dev only), the mount → cleanup → mount cycle flipped the ref to `false` permanently and never reset it, so every async fetch resolved with the guard tripped and folders/works/exercises/analyses/draft saves never reached state. Fixed by setting `mountedRef.current = true` at the top of the same effect — production builds (port `:5553`/`:5555`) were unaffected because StrictMode does not double-mount in production. Touched: `client/src/pages/WritersRoom.jsx`, `client/src/components/writers-room/{ExercisePanel,AiPanel,WorkEditor}.jsx`, `client/src/pages/Ask.jsx`.
