# Release v{NEXT}

## Added

- **Route-level tests for proof/final `target` + `useProofAsBase` on the comic page/cover render schemas.** Three new tests in `server/routes/pipeline.test.js` exercise the two-slot variant flow: cover render with `target: 'final'` + `useProofAsBase: true` lands the new in-flight job in `cover.finalImage` (with `fromProof: true` stamped on by `sanitizeRenderSlot`) while leaving the prior `cover.proofImage` slot intact; the same flow for page renders is covered against `pages[i]`; and both `comicPageRenderSchema` + `comicCoverRenderSchema` reject invalid `target` enum values with 400 (Zod rejection before service dispatch). The mocked `enqueueComicCover` / `enqueueVisualComicPage` stubs now forward `variant` + `fromProof` from the request body so the route's `slotKeyForVariant(result.variant)` decision is driven by realistic service-shape data.

## Changed

- **Universe-as-Canon Phase 2 UI — canon folded into Universe Builder.** Characters / Places / Objects are now managed inline on the Universe Builder page instead of a separate `/universe-builder/:id/canon` page. New `UniverseCanonSection` component embeds between the bible (logline/premise/styleNotes) and the composite-sheets editor, surfacing the same affordances the old page had: Extract from prose, AI differentiate cast, per-entry lock toggle, refine character, render reference, clean plates for settings, and the cross-series filter dropdown. The canon mutations write to the server immediately and only the canon arrays (`characters`/`settings`/`objects`) plus `updatedAt` are merged back into the page draft, so pending unsaved edits to logline / premise / influences aren't clobbered by a stale server copy. The `/universe-builder/:id/canon` route is preserved as a `<Navigate replace>` redirect to `/universe-builder/:id` (preserves the `?series=<id>` filter query) so deep-links and bookmarks keep working. The standalone "Canon" link button in the Universe Builder header is gone — canon is right there on the page now. Series Pipeline's "Manage characters, places, and objects on the linked Universe →" link points at the builder directly. `UniverseCanon.jsx` (548 LOC) deleted; navigation manifest's existing `canon` / `universe-canon` aliases continue to resolve to the builder, so voice "take me to canon" still works.
- **CanonCard — Render + Refine both gated by lock.** Locking a canon entry now disables both AI rewrite paths (refine, differentiate) AND new visual-ref renders (reference, clean plate). Locked entries signal frozen identity — both description and visuals are pinned. Tooltips on the disabled buttons direct the user to unlock first. Previously only refine was gated; render-reference fired regardless of lock state.
