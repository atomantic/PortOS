# Architecture Decision Records

One file per accepted decision, named `YYYY-MM-DD-<slug>.md` (date of acceptance), starting with `# ADR: <title>` and a `**Date:**` line.

ADRs record *why* a structural choice was made and what the alternatives were. They are point-in-time records — when a decision is superseded, add a new ADR that references the old one rather than editing history.
