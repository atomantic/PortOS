# Superpowers Plan/Spec Records

Artifacts from builds driven by the superpowers workflow, kept as point-in-time records:

- `specs/YYYY-MM-DD-<slug>-design.md` — the approved design document.
- `plans/YYYY-MM-DD-<slug>.md` — the matching implementation plan (headed with the subagent-driven-development skill requirement).

A spec pairs with the plan sharing its slug; some early specs (`2026-04-20-rigged-3d-avatar-stage-design`, `2026-05-10-cd-pipeline-polish-design`) have no plan file because implementation was driven directly from the spec.

New design records outside the superpowers workflow go in [`docs/plans/`](../plans/README.md) instead.
